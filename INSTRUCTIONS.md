# Instructions
**Student Project - Compulsory Activity**

## 1. Project Overview
This compulsory activity requires you to work in groups to develop a practical AI/ML application. You will design, implement, and evaluate a system that demonstrates your understanding of artificial intelligence and machine learning concepts covered in the course.

## 2. Learning Objectives
By completing this project, you will:

Apply AI/ML techniques to solve real-world problems
Implement and evaluate machine learning models
Demonstrate understanding of data preprocessing, model training, and validation
Present and communicate technical concepts effectively
Work collaboratively in a team environment
 

## 3. Group Formation
Group Size: 2 students per group (preferred)

Individual projects or groups of 3 may be considered in exceptional circumstances. Please contact if you require an alternative group size.

Registration: Register your group and chosen topic by the end of week 7 on Canvas.

## 4. Topic Selection
You may choose from the suggested topics below or propose your own topic. All custom topics must be approved.

### 4.1 Suggested Topics
#### Recommender Systems
Develop a recommendation engine for movies, music, books, or e-commerce products. Implement collaborative filtering, content-based filtering, or hybrid approaches.
___

#### Speech Recognition
Build a speech-to-text system or voice command recognition application. Explore acoustic models, language models, and modern deep learning approaches.
___

#### Natural Language Processing
Create an NLP application such as sentiment analysis, text classification, named entity recognition, chatbot, or text summarization system.
___

#### Time Series Analysis
Develop a forecasting model for stock prices, weather data, energy consumption, or sensor data. Implement traditional statistical methods or neural network-based approaches.
___

#### Image Recognition
Build an image classification, object detection, or image segmentation system. Apply convolutional neural networks (CNNs) or transfer learning techniques.
___

### 4.2 Proposing Your Own Topic
If you wish to work on a different topic, submit a brief proposal (200-300 words) including:

Problem statement and motivation
Proposed AI/ML techniques
Expected outcomes
Data sources (if applicable)
 

## 5. Project Requirements
Your project must include the following components:

### 5.1 Implementation
- Working code implementation in Python (preferred) or another suitable programming language
- Appropriate use of AI/ML libraries and frameworks (e.g., scikit-learn, TensorFlow, PyTorch)
- Well-documented and organized code
- Version control using Git (GitHub/GitLab repository)
 ___

### 5.2 Data
- Appropriate dataset(s) for your chosen problem
- Data preprocessing and cleaning procedures
- Proper train/validation/test splits
 ___

### 5.3 Evaluation
Appropriate evaluation metrics for your problem
Baseline comparisons or benchmarking
Analysis of results and model performance
 ___

## 6. Deliverables
### 6.1 Written Report (40%)
Submit a technical report (3,000-4,000 words) covering:

- Introduction and problem definition
- Related work and background
- Methodology and implementation details
- Experiments and results
- Discussion and analysis
- Conclusion and future work
- References
 ___

### 6.2 Code Repository (30%)
- Complete, well-documented source code
- README file with setup instructions and usage examples
- Requirements file (e.g., requirements.txt or environment.yml)
- Sample data or instructions to obtain data
 ___

### 6.3 Presentation (20%)
Deliver a 15-minute presentation (+ 5 minutes Q&A) covering:

- Problem motivation and objectives
- Technical approach and methodology
- Key results and findings
- Live demonstration (if applicable)
- Lessons learned and reflections
 ___

### 6.4 Individual Reflection (10%)
Each team member must submit a brief individual reflection (500-750 words) addressing:

- Your specific contributions to the project
- Challenges faced and how you overcame them
- Key learnings and skills developed
- Team collaboration experience
 ___

 ___

## 9. Submission Guidelines
- **Report**: Submit as PDF via Canvas assignment
- **Code**: Provide GitHub/GitLab repository link in Canvas. Ensure the repository is public or that the instructor has been granted access.
- **Presentation Slides**: Upload to Canvas before presentation day
 

 

## 10. Academic Integrity
- All work must be your own. Properly cite all sources, including code snippets, libraries, and datasets.
- You may use AI tools (e.g., ChatGPT, GitHub Copilot) as learning aids, but you must understand and be able to explain all code you submit.
